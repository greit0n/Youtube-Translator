1What this is / how it works

This is a two-part personal tool that overlays auto-generated English subtitles onto YouTube videos, translated live from whatever language is being spoken. It runs completely free and locally on your computer's GPU — no subscriptions, no API keys, nothing leaves your machine.

There are two pieces working together:

  • helper A small Python program (in the helper/ folder). It uses yt-dlp + ffmpeg to grab the video's audio and faster-whisper (the large-v3 model) on your GPU to transcribe the speech, then hands it to a local Ollama LLM to translate into English. It listens on http://127.0.0.1:8765.
  • extension A Chrome extension (in the extension/ folder) that draws those English subtitles over the YouTube player, perfectly synced to playback.

Two-stage translation (new default)

Translation now happens in two stages for far more faithful results:

  • Stage 1 — Transcribe. Whisper (large-v3) writes down what's actually spoken, in the original language.
  • Stage 2 — Translate. A local Ollama chat model (default qwen2.5:7b) translates that into natural English while preserving profanity, slang, and names instead of sanitizing them.

You choose this in the popup under Translation engine: Faithful (Ollama) is the new default. If Ollama isn't running, the tool automatically falls back to Whisper's own built-in translation (the old Fast mode) so you still get subtitles.

Fast start & pre-buffer

The helper no longer downloads the whole video first. Instead it fetches and translates audio in rolling windows starting from your current position, so the first subtitles appear within a few seconds (it used to take up to a minute on long videos). With the Pre-buffer toggle on (the default), it starts building subtitles the moment you open a video — before you even press play.

The first run downloads a ~3 GB Whisper model, and the first Ollama model you pull is ~5 GB. Both happen once, then load from disk and start quickly.

To keep lag near zero, the helper transcribes and translates slightly ahead of where you're watching, so subtitles are ready by the time playback reaches them.

VODs (regular videos) only for now. Live-stream support is a planned future feature.

2Prerequisites

Before you start, make sure your PC has the following. (Yours already does — this is just the checklist.)

  • Windows — you're on Windows 11. ✅
  • A CUDA-capable NVIDIA GPU — your RTX 5070 Ti is perfect for this. ✅
  • Python 3.10 or newer installed.
  • ffmpeg installed and on your PATH — now required (it's used to seek into the video and extract the audio windows).
  • Ollama installed and running — powers the new faithful translation. Get it from ollama.com.

Check Python and ffmpeg

Open a terminal (PowerShell or Command Prompt) and run:

python --version
ffmpeg -version

If both print version numbers, you're good. If python isn't recognized, install Python 3.10+ from python.org and tick “Add Python to PATH” during setup.

Installing ffmpeg on Windows

ffmpeg is required now, so don't skip this. The easiest way is with winget:

winget install Gyan.FFmpeg

Alternatively, download a build from gyan.dev, unzip it, and add its bin folder to your PATH.

After installing ffmpeg, open a fresh terminal before re-running ffmpeg -version — a new terminal is needed to pick up the updated PATH.

Installing Ollama & pulling a translation model

Download and install Ollama from ollama.com. Once installed it runs in the background. Then pull the default chat model used for translation:

ollama pull qwen2.5:7b

That's a ~5 GB download (once). On a 16 GB GPU it runs comfortably alongside Whisper. Confirm Ollama is up and the model is there:

ollama list

Your existing qwen3-embedding model will NOT work for translation. Embedding models only turn text into vectors — they can't generate translations. You need a chat model like qwen2.5:7b.

If Ollama isn't installed or running, the tool still works — it just falls back to Whisper's built-in (less faithful) translation.

3Step 1 — Set up & start the local helper

  1. Open a terminal in the helper/ folder.

    In File Explorer, navigate into the helper folder, then type cmd (or powershell) in the address bar and press Enter. The terminal opens right in that folder.

  2. Install the dependencies. (One time, or whenever they change.) 
    pip install -r requirements.txt

    This now pulls in WhisperX — a faster, smarter transcription engine that also powers the new Speaker labels feature. Everything else works exactly as before.

    Keep your GPU working: the ctranslate2 version is pinned on purpose. requirements.txt locks ctranslate2 to version 4.8.0 — the only version that runs on your RTX 5070 Ti's GPU. Don't manually upgrade it (and if something else tries to, re-run pip install -r requirements.txt to put it back). After installing, confirm the GPU is still in use: the startup logs should mention CUDA (not CPU), and a real video should produce subtitles within seconds.

  3. (Optional) Install the extras for Clean audio & Speaker labels.

    Only needed if you want background-noise/music removal or speaker labels (see New settings). For Clean audio (Light & Music modes):

    pip install -r requirements-optional.txt

    For Speaker labels you don't need any extra pip install, but you do need a free HuggingFace token (a one-time setup):

    • Create a free account at huggingface.co.
    • While logged in, open pyannote/speaker-diarization-community-1 and pyannote/embedding and click “Agree and access repository” on each (the Website field accepts any URL, e.g. your GitHub).
    • Create a Read access token under Settings → Access Tokens, then give it to the helper one of two ways: set an HF_TOKEN environment variable, or save the token on one line into a file at helper\hf_token.txt.

    These extras are completely optional. If you skip them, Clean audio and Speaker labels simply do nothing (subtitles still work normally) — nothing breaks.

  4. Start the helper. 
    python server.py

    On the very first start, this downloads the ~3 GB large-v3 model. Give it a few minutes — progress shows in the terminal.

  5. Make sure Ollama is running.

    For faithful translation, Ollama needs to be running with a chat model pulled. If you installed it (see Prerequisites), it usually starts on its own — confirm with ollama list and check qwen2.5:7b is listed. If it's not running, the helper quietly falls back to Whisper's built-in translation.

  6. Wait until it's ready.

    The helper is ready once the terminal prints that it's listening on 127.0.0.1:8765.

Leave this terminal running the whole time you're watching videos. Closing it stops the helper, and subtitles will stop working. Minimize it and forget about it.

Want to confirm your GPU is being used? Watch the startup logs — they'll mention CUDA when the GPU is active (rather than CPU).

4Step 2 — Install the Chrome extension

You only need to do this once.

  1. Open the extensions page. In Chrome, go to: 
    chrome://extensions
  2. Enable Developer mode using the toggle in the top-right corner.
  3. Click “Load unpacked.” A button appears on the left after Developer mode is on.
  4. Select the extension/ folder from this project. The extension installs immediately and its icon appears in your toolbar.

Pin the extension to your toolbar (click the puzzle-piece icon → pin) so it's always one click away while watching.

5Step 3 — Watch a video

With the helper running (Step 1) and the extension installed (Step 2), you're ready to go.

  1. Open any YouTube video.
  2. Click the extension icon to open its settings popup.

    You should see a green ● Connected status — that confirms the popup found your running helper.

  3. Make sure it's set to “Enabled.”
  4. Leave the language on “Auto” so it detects the spoken language automatically — or pick one manually (e.g. Czech) if you prefer.
  5. Pick a Translation engine.

    Leave it on Faithful (Ollama) (the default) for the most accurate, uncensored translations. Fast (Whisper built-in) skips Ollama and uses Whisper's own translation — quicker, but it tends to clean up slang and profanity.

  6. Choose the Ollama model.

    Use the Ollama model dropdown to pick the chat model for translation — qwen2.5:7b is the recommended default. If the dropdown is empty, you haven't pulled a model yet: run ollama pull qwen2.5:7b and reopen the popup.

  7. Leave Pre-buffer on.

    With Pre-buffer on (the default), subtitles start building the moment the video page loads, so they're ready almost instantly when you hit play.

  8. Pick a caption size that's comfortable to read.
  9. Press play. English subtitles appear overlaid on the player, synced to playback. 🎉

If the status shows not connected, the helper isn't running — jump back to Step 1 and start python server.py.

Hit a 🔒 “Age-restricted video needs your YouTube login” error? That video needs your YouTube login — see Age-restricted videos & login (cookies.txt) for the one-time fix.

6New settings

The popup has four newer settings. All are optional — the defaults are sensible, so you can ignore these until you want them. Changing any of them restarts the current video's subtitles automatically.

Quality

Picks the Whisper model based on your hardware. Bigger models are more accurate but use more GPU memory and run a touch slower.

  • Auto (default) — the helper checks your GPU's memory and picks the best model that fits: large-v3 on a big GPU (like your RTX 5070 Ti), stepping down to medium / small / base on smaller ones.
  • Max — always large-v3 (highest quality).
  • Balanced — medium (good quality, lighter).
  • Lite — small (fastest, least accurate).

On your machine, Auto and Max both land on large-v3. Switching this can cause a brief “switching model” pause while the new model loads.

Clean audio

Removes background noise or music before transcription, which can sharpen the subtitles on noisy footage. Requires the optional extras (pip install -r requirements-optional.txt — see Step 1); without them this safely does nothing.

  • Off (default) — no processing, lowest latency.
  • Light — gentle background-noise suppression (DeepFilterNet). Good for hiss, wind, room noise.
  • Music — isolates voices from background music (Demucs). The most thorough, but the heaviest — expect a little extra delay.

Speaker labels

Tags each line with a stable “Speaker 1”, “Speaker 2”… label (and a distinct color) so multi-person videos are easier to follow. Off by default. Needs the free HuggingFace token from Step 1; without it, this simply stays off and subtitles look normal. It adds a little processing time per window.

Glossary

A box where you can list names and terms the tool should get right — one per line. This both helps Whisper hear the word correctly and keeps the translation consistent. Two formats:

Zelensky
Stalingrad
favela = slum

A plain line (Zelensky) just biases recognition toward that spelling. A term = preferred line (favela = slum) also tells the translator which English wording you want. Leave it empty if you don't need it.

7Age-restricted videos & login (cookies.txt)

Most YouTube videos work with no setup at all. But age-restricted videos — and, occasionally, a YouTube “confirm you're not a bot” check — require your YouTube login to download. When this happens, the helper shows an error like:

🔒 Age-restricted video needs your YouTube login. Add a cookies.txt

The fix is to give the helper a one-time export of your YouTube login as a cookies.txt file.

Why a file, and not just “read my Chrome login”? On Windows, software can't read Chrome's cookies directly anymore — Chrome 127+ encrypts them with App-Bound Encryption, and the cookie database is locked while Chrome is running. So instead you export a cookies.txt from the browser with a small extension.

Set it up (one time)

  1. Install the “Get cookies.txt LOCALLY” extension.

    In Chrome, open the Chrome Web Store and add the extension named exactly “Get cookies.txt LOCALLY”. It's open-source and reads cookies locally through Chrome's extension API — it never uploads them. (Make sure you pick the one named LOCALLY.)

  2. Be logged into YouTube in Chrome.

    Use an account that's 18+ for age-restricted content. Chrome can stay open the whole time.

  3. Export the cookies.

    Open a youtube.com tab, click the extension's icon, and click Export. Set its format to “Netscape” / cookies.txt; if it offers a “Current Site” option while you're on youtube.com, that's enough. It downloads a cookies.txt file.

  4. Save the file into the helper folder.

    Move/save the downloaded file to exactly this path:

    C:\Users\georg\Projects\Youtube-Translator\helper\cookies.txt
  5. That's it — no restart needed.

    The helper auto-detects the file. Reload the YouTube tab and play the age-restricted video. The extension popup status will show “Cookies: ✓” once the helper detects the file.

The helper auto-detects helper\cookies.txt. You can also point it at a file elsewhere by setting the YTDLP_COOKIES_FILE environment variable to that file's full path.

Cookies expire / rotate. If a video later says “Sign in to confirm your age” again, just re-export cookies.txt (repeat steps 3–4). For maximum stability, export from an Incognito window: log into YouTube in Incognito, export, then close the Incognito window so the session isn't rotated out from under the file.

The cookies.txt file contains your YouTube login tokens — treat it like a password. Keep it private; it stays on your machine and is never uploaded.

Full age-restricted setup (PO token + JS challenge)

For age-restricted (18+) videos, cookies.txt alone is no longer enough. YouTube now also demands a proof-of-origin (PO) token and a solved JavaScript challenge before it will hand over the audio streams. Normal videos need none of this — this whole stack only matters for the "needs_auth" age-restricted case. It's a one-time setup; once in place it's automatic.

There are three pieces:

  1. cookies.txt — passes the age gate.

    Already covered above. Without it you'll get "Sign in to confirm your age".

  2. A PO-token server, run via Docker.

    Docker Desktop must be installed and running. Then start the token provider once — it auto-restarts with Docker and listens on 127.0.0.1:4416:

    docker run -d --name bgutil-pot --restart unless-stopped -p 4416:4416 brainicism/bgutil-ytdlp-pot-provider

    Check it's alive by opening http://127.0.0.1:4416/ping — it should return JSON. The matching yt-dlp plugin is already installed; if you ever need to (re)install it:

    pip install -U bgutil-ytdlp-pot-provider
  3. A JavaScript challenge solver (YouTube's "n challenge").

    Install the yt-dlp EJS plugin and the Deno runtime (yt-dlp's preferred JS engine — Node alone is not used):

    pip install -U yt-dlp-ejs
winget install DenoLand.Deno

    Then restart the helper / open a new terminal so Deno is on your PATH.

This stack only matters for age-restricted videos. Once set up it's automatic and persists — the Docker container auto-restarts with Docker, so there's nothing to start by hand. The popup shows "Cookies: ✓" when cookies are detected.

Diagnosing which piece is missing (read the helper's error text):

"Sign in to confirm your age" → you need cookies (step 1).
"Requested format is not available" / only storyboard formats offered → you need the PO-token Docker server (step 2).
"n challenge solving failed" / "JS runtimes: none" → you need Deno + yt-dlp-ejs (step 3).

If you have YouTube Premium, non-age-restricted videos generally don't need the PO token. But age-restricted ("needs_auth") videos still require the full stack above — Premium does not exempt them.

8Troubleshooting

ProblemFix
No subtitles / “helper not running” Make sure python server.py is running in the helper/ folder and that the popup shows Connected. The helper must stay open while you watch.
Subtitles slow to appear on first play The model is probably still downloading or loading. Check the helper terminal — once it says it's listening on 127.0.0.1:8765, subtitles will start flowing.
Status shows CPU instead of CUDA Your GPU isn't being used, so it'll be slow. This means the CUDA / PyTorch GPU setup isn't active. Reinstall a CUDA-enabled build of torch and confirm your NVIDIA drivers are up to date, then restart the helper.
Went to CPU after a reinstall / update Something likely upgraded ctranslate2 off the pinned 4.8.0, which breaks GPU support on your card. Restore the pin by re-running pip install -r requirements.txt in the helper/ folder, then restart the helper.
Clean audio / Speaker labels seem to do nothing These need their extras. For Clean audio, run pip install -r requirements-optional.txt. For Speaker labels, set up the free HuggingFace token (see Step 1). Without them, both quietly stay off and normal subtitles still work.
Subtitles out of sync after seeking If you scrub/jump around in the video and captions drift, simply reload the page to resync.
Translations look censored / not faithful Make sure the Translation engine in the popup is set to Faithful (Ollama) (not Fast), that you've run ollama pull qwen2.5:7b, and that Ollama is actually running (ollama list). The Fast/Whisper engine tends to sanitize slang and profanity.

9Notes & future

  • Free, local & private. Everything runs on your own machine. The audio is processed locally and never leaves your PC — no cloud services, no API keys, no costs.
  • VOD-only for now. Regular YouTube videos work today; live streams aren't supported yet.
  • Live-stream support is planned as a future feature.
  • Possible open-source release. This may eventually be polished up and released as a free, open-source extension.

Daily routine, in short: make sure Ollama is running → start the helper (python server.py) → open a YouTube video → enjoy the subtitles. That's it.
