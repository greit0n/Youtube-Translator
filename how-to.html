YouTube English Subtitles

Overlay AI-generated English subtitles on YouTube videos using a local Chrome extension and a local Python helper. No hosted backend, no paid APIs, and no cloud transcription service.

1. What this is

This project has two parts:

  • extension/: a Chrome MV3 extension that draws subtitles over the YouTube player.
  • helper/: a local FastAPI server on 127.0.0.1:8765 that resolves YouTube audio, transcribes it with WhisperX, optionally translates it with Ollama, and streams subtitle segments back to the extension.

The helper fetches short audio windows around the current playhead instead of downloading the whole video. That rolling-window design is what makes captions appear quickly on long VODs.

Default workflow: the current best-tested path is Czech source speech to English using WhisperX plus Ollama gemma2:9b. Fast direct Whisper translation is also available and does not require Ollama.

2. Prerequisites

  • Chrome or another Chromium browser that can load unpacked extensions.
  • Python 3.10 or newer.
  • ffmpeg installed and available on PATH.
  • A CUDA-capable NVIDIA GPU is recommended. CPU fallback exists, but it is slower.
  • Ollama with a chat model is recommended for the default translation mode.
python --version
ffmpeg -version

Install ffmpeg on Windows with:

winget install Gyan.FFmpeg

Install Ollama from ollama.com, then pull the recommended model:

ollama pull gemma2:9b
ollama pull aya-expanse:8b

3. Start the helper

From the project root:

cd helper
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python server.py

The first run downloads the selected Whisper model. The model loads in the background, so `/health` may report model_loaded:false until the download and load finish.

Do not loosen the ctranslate2==4.8.0 pin. It is required for the tested RTX 5070 Ti / Blackwell CUDA path. After dependency changes, confirm `/health` reports "cuda": true if you expect GPU inference.

Optional clean-audio extras:

pip install -r requirements-optional.txt

Speaker labels require a free HuggingFace token. Provide it with HF_TOKEN, HUGGINGFACE_TOKEN, or a local helper\hf_token.txt file. Never commit that token file.

4. Install the extension

  1. Open chrome://extensions.
  2. Enable Developer mode.
  3. Click Load unpacked.
  4. Select this repository's extension/ folder.
  5. Pin the extension icon if you want easy access to the popup.

After editing extension files, reload the unpacked extension and hard-reload the YouTube tab.

5. Watch a video

  1. Start the helper with python server.py.
  2. Open a YouTube video.
  3. Open the extension popup.
  4. Confirm the helper status is connected.
  5. Enable subtitles on that tab.
  6. Leave Pre-buffer enabled for the quickest first captions.

Each YouTube tab is opt-in. Disabling the toggle tears down that tab's WebSocket and removes the overlay.

6. Settings

Translation engine

  • Accurate Czech: WhisperX transcribes Czech source speech, then Ollama translates to English with glossary support.
  • Fast Whisper: Whisper translates directly to English without Ollama.

Quality

  • Auto: choose the best model that fits detected VRAM.
  • Max: force large-v3.
  • Balanced: use medium.
  • Lite: use small.

Clean audio

Optional denoise or vocal isolation before transcription. Requires `requirements-optional.txt`; otherwise it safely passes audio through.

Speaker labels and enrolled-only mode

Speaker labels need a HuggingFace token. Enrolled-only mode also needs a reference clip in helper\enroll\. If requirements are missing, the helper warns and shows all speakers.

Glossary

Use one entry per line, either term or term = preferred translation. The Ollama path injects these terms into the prompt for consistency.

7. Cookies and age-restricted videos

Most public videos can be tried without cookies, but authenticated cookies often make YouTube range seeking more reliable. Age-restricted videos require a YouTube login export.

  1. Install a reputable browser extension that exports cookies in Netscape cookies.txt format, such as "Get cookies.txt LOCALLY".
  2. Log into YouTube in the browser.
  3. Export YouTube cookies in Netscape format.
  4. Save the file as helper\cookies.txt, or set YTDLP_COOKIES_FILE to another full path.

cookies.txt contains login tokens. Treat it like a password. It stays local and must never be committed or shared.

Full age-restricted stack

Some age-restricted videos also require a proof-of-origin token provider and a JavaScript challenge solver:

docker run -d --name bgutil-pot --restart unless-stopped -p 4416:4416 brainicism/bgutil-ytdlp-pot-provider
pip install -U bgutil-ytdlp-pot-provider
pip install -U yt-dlp-ejs
winget install DenoLand.Deno

The token provider listens on 127.0.0.1:4416. Open a new terminal after installing Deno so it is available on PATH.

8. Troubleshooting

ProblemFix
Helper not connected Run python server.py from helper/ and keep that terminal open.
First subtitles are slow The Whisper model may still be downloading or loading. Check `/health` and the helper terminal.
CPU instead of CUDA Check NVIDIA drivers, CUDA-enabled PyTorch, and the pinned ctranslate2==4.8.0 dependency.
No Ollama model in popup Run ollama pull gemma2:9b, confirm with ollama list, then reopen the popup.
Age-restricted login error Export YouTube cookies to helper\cookies.txt. If needed, set up the PO-token and Deno stack above.
yt-dlp extraction errors Update yt-dlp with pip install -U yt-dlp and try again.
